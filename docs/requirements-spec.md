## 1. 项目概述

### 1.1 项目目标

在 Raspberry Pi 5 上部署全离线双语语音交互系统，实现中英文之间的语音识别、文本翻译、语音合成功能，并提供历史记录管理和录音导出能力。

### 1.2 系统边界

- 全离线运行，不依赖任何网络连接
- 仅支持双语（语言 A / 语言 B，默认中文 / 英文）
- 运行于 Raspberry Pi 5 单板计算机
- 通过 ReSpeaker 2-Mics Pi HAT V2 采集音频

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 语言 A | 第一语言（默认：中文） |
| 语言 B | 第二语言（默认：英文） |
| ASR | Automatic Speech Recognition，自动语音识别 |
| MT | Machine Translation，机器翻译 |
| TTS | Text-to-Speech，文本转语音 |
| Pipeline | ASR/MT/TTS 模块的串联处理链路 |

## 2. 功能需求

### FR-01: A 文字输入转 A 语音输出 (TTS)

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | TTS |
| 触发方式 | 用户通过接口提交 A 语言文本 |

- 接收 A 语言文本输入
- 调用 TTS 引擎合成 A 语言语音
- 通过扬声器播放合成语音
- 同时将合成音频保存为 WAV 文件（供历史记录使用）

验收标准:
- 输入任意 A 语言文本（长度 <= 500 字符），系统在 3 秒内开始播放语音
- 合成语音可被人类听懂

### FR-02: B 语音输入转 B 文字输出 (ASR)

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | ASR |
| 触发方式 | 用户通过接口触发录音或提交音频文件 |

- 通过麦克风采集 B 语言语音输入
- 调用 ASR 引擎识别为 B 语言文本
- 返回识别结果文本

验收标准:
- 安静环境下，标准语速 B 语言语音识别准确率 >= 80%
- 单句识别延迟 < 5 秒

### FR-03: A 文字输入转 B 语音输出 (MT + TTS)

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | MT, TTS |
| 触发方式 | 用户通过接口提交 A 语言文本 |

- 接收 A 语言文本输入
- 调用 MT 引擎翻译为 B 语言文本
- 调用 TTS 引擎合成 B 语言语音
- 播放合成语音并保存音频文件

验收标准:
- 翻译结果语义基本正确（简单日常用语）
- 端到端延迟（输入文本到开始播放）< 5 秒

### FR-04: B 语音输入转 A 文字输出 (ASR + MT)

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | ASR, MT |
| 触发方式 | 用户通过接口触发录音或提交音频文件 |

- 通过麦克风采集 B 语言语音输入
- 调用 ASR 引擎识别为 B 语言文本
- 调用 MT 引擎翻译为 A 语言文本
- 返回翻译结果文本

验收标准:
- 端到端延迟（录音结束到返回文本）< 8 秒
- 翻译结果语义基本正确（简单日常用语）

### FR-05: 历史记录管理

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | Storage |
| 存储上限 | 最近 5 条 |

- 每次语音交互完成后，自动保存交互记录
- 每条记录包含：时间戳、功能类型（FR-01~04）、音频文件路径、文本内容（源文本和/或翻译文本）
- 仅保留最近 5 条记录，超出时自动删除最早的记录及其关联音频文件
- 支持查询历史记录列表
- 支持通过导出接口将历史记录（音频 + 文本元数据）导出至其他设备

验收标准:
- 记录数量始终 <= 5，超出时最早记录被正确删除
- 导出接口可被同一局域网内的设备访问并下载文件

### FR-06: 录音功能

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模块 | Audio, Storage |
| 单次上限 | 3 分钟 |
| 文件上限 | 5 个 |

- 独立于翻译功能的纯录音模式
- 单次录音时间上限 3 分钟，到达上限自动停止并保存
- 支持手动停止录音
- 最多保存 5 个录音文件，超出时自动删除最早的录音
- 录音文件格式：16kHz, 16-bit, Mono WAV
- 支持通过导出接口将录音文件导出至其他设备

验收标准:
- 录音到达 3 分钟时自动停止，文件完整可播放
- 录音文件数量始终 <= 5
- 导出接口可正常下载录音文件

## 3. 非功能需求

### 3.1 离线运行 (NFR-01)

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 类型 | 约束 |

- 系统所有功能必须在无网络连接的环境下正常运行
- 所有模型文件、语言包预先部署到本地存储
- 不得调用任何远程 API 或云服务
- 导出接口仅在局域网内提供服务（非互联网依赖）

### 3.2 性能要求 (NFR-02)

| 指标 | 目标值 | 说明 |
|------|--------|------|
| TTS 延迟 | < 3 秒 | 文本输入到开始播放 |
| ASR 延迟 | < 5 秒 | 录音结束到返回文本 |
| MT 延迟 | < 3 秒 | 文本输入到返回翻译 |
| 端到端延迟（FR-03/04） | < 8 秒 | 输入到最终输出 |
| ASR 准确率 | >= 80% | 安静环境、标准语速 |
| 系统启动时间 | < 30 秒 | 从开机到可接受输入 |

### 3.3 资源限制 (NFR-03)

| 资源 | 限制 |
|------|------|
| 模型总内存占用 | < 2 GB |
| 系统总内存占用 | < 4 GB（含 OS） |
| 模型文件总磁盘占用 | < 2 GB |
| 运行时数据磁盘占用 | < 500 MB（历史记录 + 录音） |
| CPU 使用率（空闲时） | < 10% |

### 3.4 可靠性 (NFR-04)

- 系统应能连续运行 24 小时无崩溃
- ASR 识别失败时返回明确错误信息，不导致系统中断
- 翻译失败时返回原文并标注错误
- 磁盘空间不足时拒绝新录音并提示用户

### 3.5 可维护性 (NFR-05)

- 模块化设计：ASR、MT、TTS、Audio、Storage 各模块独立，可单独替换
- 配置外部化：模型路径、音频参数、存储路径等通过配置文件管理
- 日志记录：使用标准 `logging` 模块记录关键操作和错误，便于调试

## 4. 硬件需求

### 4.1 目标平台

| 组件 | 规格 |
|------|------|
| 主板 | Raspberry Pi 5 |
| CPU | Broadcom BCM2712, 4-core Cortex-A76 @ 2.4GHz |
| RAM | 8 GB LPDDR4X |
| 存储 | microSD 卡 >= 32 GB（推荐 64 GB） |
| OS | Raspberry Pi OS (Debian Bookworm, ARM64) |

### 4.2 音频硬件

| 组件 | 规格 |
|------|------|
| 麦克风 | ReSpeaker 2-Mics Pi HAT V2 (Seeed Studio) |
| 接口 | I2S 数字音频接口 |
| 麦克风数量 | 双麦克风阵列 |
| 扬声器 | 3.5mm 音频输出或 HAT 板载扬声器接口 |

### 4.3 驱动要求

- 安装 Seeed 提供的设备树覆盖文件 (`seeed-2mic-voicecard`)
- 在 `/boot/config.txt` 中加载 `dtoverlay=seeed-2mic-voicecard`
- ALSA 驱动正确配置，`arecord -l` 和 `aplay -l` 可见声卡设备
- 默认声卡配置写入 `/etc/asound.conf` 或 `~/.asoundrc`

## 5. 技术约束

### 5.1 技术选型

| 模块 | 首选方案 | 备选方案 | 说明 |
|------|----------|----------|------|
| ASR | Vosk | Whisper-tiny | Vosk 模型小 (~50MB)、延迟低；Whisper-tiny (~39MB) 多语言支持更好但延迟较高 |
| MT | Argos Translate | - | 基于 OpenNMT，支持离线中英翻译，模型 ~100-300MB |
| TTS | piper-tts | eSpeak-ng | piper-tts 语音质量更自然；eSpeak-ng 体积最小但发音机械 |
| 音频采集 | pyalsaaudio | PyAudio | ALSA 原生绑定，与 ReSpeaker HAT 兼容性好 |
| 音频播放 | aplay (subprocess) | pyalsaaudio | 系统命令调用简单可靠 |
| Web 框架 | Flask | - | 轻量级，仅用于导出接口 |
| 开发语言 | Python 3.11+ | - | 所有模块统一使用 Python |

### 5.2 音频格式标准

| 参数 | 值 |
|------|-----|
| 采样率 | 16000 Hz |
| 位深 | 16-bit |
| 通道数 | Mono (单声道) |
| 文件格式 | WAV (PCM) |

### 5.3 模型部署约束

- 所有模型文件预先下载并部署到 Pi5 本地 `models/data/` 目录
- 模型训练/微调在笔记本或更强机器上完成，Pi5 仅做推理
- Vosk 模型：下载 `vosk-model-small-cn` (中文) 和 `vosk-model-small-en` (英文)
- Argos 模型：下载 `en-zh` 和 `zh-en` 语言对的 `.argosmodel` 文件
- TTS 模型：根据选型下载对应语音包

## 6. 接口需求

### 6.1 内部模块接口

#### ASR 模块接口

```
输入: 音频文件路径 (str) 或音频字节流 (bytes)
输出: 识别文本 (str)
异常: ASRError (识别失败时抛出)
```

#### MT 模块接口

```
输入: 源文本 (str), 源语言代码 (str), 目标语言代码 (str)
输出: 翻译文本 (str)
异常: TranslationError (翻译失败时抛出)
```

#### TTS 模块接口

```
输入: 文本 (str), 语言代码 (str)
输出: 音频文件路径 (str)
异常: TTSError (合成失败时抛出)
```

#### Audio 模块接口

```
录音:
  输入: 最大时长 (int, 秒), 输出路径 (str)
  输出: 实际录音文件路径 (str)
  约束: 最大时长 <= 180 秒

播放:
  输入: 音频文件路径 (str)
  输出: None
```

### 6.2 导出接口 (HTTP API)

导出接口通过 Flask 提供，仅在局域网内可访问。

| 端点 | 方法 | 说明 |
|------|------|------|
| `GET /api/history` | GET | 返回历史记录列表 (JSON) |
| `GET /api/history/<id>/audio` | GET | 下载指定历史记录的音频文件 |
| `GET /api/history/export` | GET | 打包下载所有历史记录 (ZIP) |
| `GET /api/recordings` | GET | 返回录音文件列表 (JSON) |
| `GET /api/recordings/<id>/audio` | GET | 下载指定录音文件 |
| `GET /api/recordings/export` | GET | 打包下载所有录音文件 (ZIP) |
| `GET /api/health` | GET | 系统健康检查 |

#### 历史记录 JSON 结构

```json
{
  "id": 1,
  "timestamp": "2026-04-07T10:30:00",
  "type": "asr_mt",
  "source_text": "Hello world",
  "target_text": "你好世界",
  "audio_file": "history_001.wav",
  "source_lang": "en",
  "target_lang": "zh"
}
```

## 7. 数据存储需求

### 7.1 存储结构

```
data/
├── history/              # 历史记录
│   ├── metadata.json     # 历史记录元数据 (最多 5 条)
│   ├── history_001.wav   # 音频文件
│   ├── history_002.wav
│   └── ...
└── recordings/           # 录音文件
    ├── metadata.json     # 录音元数据 (最多 5 条)
    ├── rec_001.wav       # 录音文件
    ├── rec_002.wav
    └── ...
```

### 7.2 元数据格式

历史记录元数据 (`data/history/metadata.json`):

```json
{
  "records": [
    {
      "id": 1,
      "timestamp": "2026-04-07T10:30:00",
      "type": "tts|asr|asr_mt|mt_tts",
      "source_lang": "zh|en",
      "target_lang": "zh|en|null",
      "source_text": "原始文本",
      "target_text": "翻译文本或null",
      "audio_file": "history_001.wav"
    }
  ],
  "max_records": 5
}
```

录音元数据 (`data/recordings/metadata.json`):

```json
{
  "recordings": [
    {
      "id": 1,
      "timestamp": "2026-04-07T10:35:00",
      "duration_seconds": 45,
      "file_name": "rec_001.wav",
      "file_size_bytes": 1440044
    }
  ],
  "max_recordings": 5
}
```

### 7.3 存储策略

- 使用 JSON 文件存储元数据（轻量，无需数据库依赖）
- FIFO 淘汰策略：新记录超出上限时，删除最早的记录及其音频文件
- 文件命名：递增编号 + 时间戳，避免冲突
- 音频文件与元数据保持一致性：删除记录时同步删除音频文件

## 8. 系统架构概览

```
┌──────────────────────────────────────────────────┐
│                  主控逻辑 (main.py)                │
│         模式选择: FR-01 / FR-02 / FR-03 / FR-04   │
│         + 录音模式 (FR-06)                         │
├──────────┬───────────┬───────────┬───────────────┤
│  Audio   │   ASR     │    MT     │     TTS       │
│  模块    │   模块    │   模块    │     模块      │
│          │           │           │               │
│ 录音/播放│ Vosk /    │  Argos    │ piper-tts /   │
│ ALSA     │ Whisper   │ Translate │ eSpeak-ng     │
├──────────┴───────────┴───────────┴───────────────┤
│              Storage 模块                         │
│     历史记录管理 (5条) + 录音管理 (5个)            │
├──────────────────────────────────────────────────┤
│              Flask 导出接口                        │
│     GET /api/history/* + GET /api/recordings/*    │
├──────────────────────────────────────────────────┤
│              硬件层                                │
│  ReSpeaker 2-Mics HAT V2 + ALSA + 扬声器         │
└──────────────────────────────────────────────────┘
```

### 数据流

| 功能 | 数据流 |
|------|--------|
| FR-01 | A 文本 --> TTS(A) --> A 语音输出 |
| FR-02 | B 语音输入 --> ASR(B) --> B 文本输出 |
| FR-03 | A 文本 --> MT(A->B) --> TTS(B) --> B 语音输出 |
| FR-04 | B 语音输入 --> ASR(B) --> MT(B->A) --> A 文本输出 |
| FR-05 | 交互结果 --> Storage(FIFO, max=5) --> 导出接口 |
| FR-06 | 麦克风 --> Audio(max=180s) --> Storage(FIFO, max=5) --> 导出接口 |

## 10. 约束与假设

### 10.1 约束

1. 全离线：系统部署完成后不依赖任何网络连接
2. 硬件固定：目标硬件为 Raspberry Pi 5 (8GB) + ReSpeaker 2-Mics HAT V2
3. 双语限定：当前版本仅支持中文和英文
4. 存储上限：历史记录 5 条、录音文件 5 个，超出自动淘汰
5. 录音时长：单次录音不超过 3 分钟
6. 模型仅推理：Pi5 上不进行模型训练，仅加载预训练模型做推理

### 10.2 假设

1. ReSpeaker 2-Mics HAT V2 驱动可在 Pi5 (Bookworm) 上正常编译安装
2. Vosk small 模型在 Pi5 上的识别延迟可控制在 5 秒内
3. Argos Translate 中英翻译模型可在 8GB RAM 的 Pi5 上正常加载
4. 使用环境为室内安静环境，背景噪声较低
5. 用户通过命令行或简单 Web UI 与系统交互

### 10.3 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ReSpeaker HAT V2 驱动不兼容 Pi5 内核 | 无法采集音频 | 备选 USB 麦克风方案 |
| Vosk 中文识别准确率不足 | FR-04 翻译质量差 | 切换 Whisper-tiny 备选方案 |
| Argos 中英翻译质量不佳 | FR-03/04 输出不可用 | 预处理优化（分句、去标点）；评估 LibreTranslate |
| eSpeak 中文发音不自然 | FR-01/03 用户体验差 | 切换 piper-tts 或 Pico TTS |
| 模型总内存超出限制 | 系统 OOM 崩溃 | 优先使用 small 模型；按需加载/卸载模型 |
