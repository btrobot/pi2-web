## 布局设计说明（遵循 Web 最佳实践）

### 1. 顶部导航栏（Header）
- **全局功能**：Logo、语言切换、用户设置、帮助入口。
- **响应式行为**：在窄屏下折叠为汉堡菜单。

### 2. 左侧导航栏（Sidebar）
- **分组菜单**：按 `Menu.py` 中的六大功能 + 录音 + 历史记录组织。
- **视觉反馈**：当前选中的菜单项高亮显示（示例中用 `◆` 标记）。
- **可折叠**：允许用户收缩以腾出更多内容空间。

### 3. 主内容区（Main Content）
分为三个垂直区域：

#### a. 当前位置面包屑
- 明确显示当前功能路径（例如：`同语言文字转语音 > 中文文字转中文语音`），降低用户迷路风险。

#### b. 输入区（Input Area）
- **文本输入框**：支持多行文本。
- **文件上传按钮**：支持上传 `.txt` 文件（自动填充内容）。
- **清空按钮**：一键清除输入。
- **控制按钮**：`开始转换`（调用后端 API）、`重置`（恢复默认）、`保存输入`（存入历史输入文件夹）。

#### c. 输出区（Output Area）
- **结果显示**：若为文本，直接显示；若为语音，显示音频播放器。
- **音频播放器**：带进度条（时长显示、拖拽、播放/暂停）。
- **操作按钮**：`下载音频`（若输出为音频）、`复制文本`（若输出为文本）、`重新转换`（快速再用相同输入转换一次）。

#### d. 最近历史记录（折叠面板）
- 默认展示最近 3 条转换记录，点击可快速重新加载输入/输出。
- 提供“查看全部历史”链接，跳转到专门的历史记录管理页面（对应 `Menu.py` 中的“历史记录管理”）。

### 4. 底部状态栏（Footer）
- 显示全局提示（如文件大小限制、支持格式）、系统状态（如 API 连接正常/异常）。

## 关键交互流程（与 Menu.py 对应）

| 菜单功能路径（Menu.py）                | Web 界面操作流程                                                                 |
|---------------------------------------|---------------------------------------------------------------------------------|
| 1.1 中文文字→中文语音                  | 左侧点击“文字→语音 > 中文→中文” → 右侧输入区输文本 → 点击“开始转换” → 输出区出现音频播放器 |
| 2.1 中文语音→中文文字                  | 左侧点击“语音→文字 > 中文→中文” → 输入区改为“上传音频文件或录音” → 上传后点击转换 → 输出区显示文本 |
| 3.1 中文文字→英文语音                  | 左侧点击“翻译+语音 > 中文→英文” → 输入中文文本 → 转换后输出英文语音播放器            |
| 6.1 中文语音→英文语音                  | 左侧点击“语音互译 > 中→英” → 输入区录音或上传中文音频 → 转换后输出英文音频播放器      |
| 2. 录音                               | 左侧点击“录音” → 主内容区显示录音控件（开始/停止，3分钟倒计时） → 录音后自动保存并可立即用于转换 |
| 3. 历史记录管理                       | 左侧点击“历史记录” → 进入独立页面，展示翻译记录和录音记录，支持播放/查看/删除        |

## 设计优势（最佳实践体现）

1. **减少菜单深度**：所有主要功能通过侧边栏一键访问，无需逐级返回。
2. **上下文相关操作**：输入区、控制区、输出区始终可见，操作流顺畅。
3. **即时反馈**：转换过程显示进度（如“语音合成中…”），完成后自动填充输出区。
4. **历史快捷复用**：最近记录折叠面板让用户快速重复或修改之前的任务。
5. **响应式预留**：布局采用 Flex/Grid，移动端可改为顶部导航 + 底部 Tab 栏。
6. **可扩展性**：新增语言对（如日韩）只需在侧边栏增加菜单项，不影响整体布局。

## 技术实现建议（非框图部分）

- **前端**：React / Vue 3 + TailwindCSS（快速实现上述布局）。
- **状态管理**：Pinia / Redux（管理当前选中的功能、输入输出数据）。
- **API 调用**：封装 TTS/ASR/MT 后端服务（可使用 WebSocket 实时返回结果）。
- **录音**：`MediaRecorder API`，限制时长 3 分钟。
- **音频播放**：`<audio>` 元素 + 自定义进度条（如 `wavesurfer.js`）。
